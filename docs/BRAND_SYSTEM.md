# BRAND_SYSTEM

Purpose:
Define the current brand system for PETTODO so that product, landing page, design, copy, and future collaborators use a consistent visual and verbal direction.

What belongs here:
- brand definition
- brand promise
- positioning
- tone
- visual direction
- logo system
- color system
- logo usage rules
- visual mistakes to avoid
- guidance for product and marketing surfaces

Update rule:
Update when the official logo, core color system, brand promise, or primary visual identity changes.

---

## 1. Brand definition

PETTODO is a community-powered network for managing and recovering pets.

It is not only a lost-pet app.
It is a practical system designed to help people care for animals, build useful animal identity over time, and respond better when a pet is lost, found, sighted, or needs coordinated support.

The brand must express:
- trust
- care
- usefulness
- safety
- practical technology
- urban/community relevance

The brand must not feel:
- childish
- overly veterinary
- NGO-like in an improvised way
- generic pet-app cliché
- centered only on AI
- centered only on lost pets

---

## 2. Brand promise

**PETTODO facilitates the care of pets and strengthens the community that protects them.**

This is the core brand promise for current use.
It combines:
- care
- utility
- protection
- community

---

## 3. Positioning

PETTODO should be understood primarily as:

- a community network for the care and safety of pets
- a practical platform for managing and recovering animals
- a useful digital layer for urban animal identity and coordination

Important:
PETTODO is **not** positioned as:
- only a pet finder
- only a pet manager
- only a service marketplace
- only a veterinary helper
- only an AI product

---

## 4. Brand personality

PETTODO should feel like a combination of:

### A. Trustworthy / structured / serious
Used when dealing with:
- recovery
- animal health-sensitive information
- community dogs
- reports
- public trust
- safety
- evidence-backed actions

### B. Warm / human / supportive
Used when dealing with:
- daily care
- education
- guidance
- community participation
- pet owners’ everyday use
- non-sensitive interactions

### Brand ratio
The brand should lean slightly more toward **trust and structure** than toward softness.

A useful mental model:
- 70% trustworthy / structured
- 30% warm / human

---

## 5. Brand strategy principles

### 5.1 PETTODO is useful before emergencies
The brand must not communicate that PETTODO is only useful when a pet gets lost.
It must communicate value in everyday care, identity, and preparation.

### 5.2 Community is important, but not chaotic
The brand should suggest community support and collective care, but not visual disorder or “crowd energy.”

### 5.3 Technology is useful, not flashy
Technology should feel purposeful and reliable.
The brand should not lean into obvious futuristic or “AI hype” aesthetics.

### 5.4 The product should feel real
The brand must support a real, useful web app — not an abstract concept, not a toy app, not a charity poster.

## 7. Color system

## 7.1 Current selected color direction

The brand direction uses a **blue + green** system.

### Color roles
- **Blue** = trust, utility, system, structure, confidence
- **Green** = care, wellbeing, support, animal care
- **White / light neutrals** = clarity, breathing room, cleanliness
- **Red** = alert/state color only, not primary brand color

### Important brand principle
Red must not dominate the brand system.
Emergency logic exists in the product, but PETTODO should not feel alarmist by default.

## 7.2 Current approved direction
The current approved logo direction uses:
- a strong **trust blue**
- a strong **care green**

Important:
Earlier exploration used `Trust Blue #2A6BB8` and `Care Green #A9E5CC` as reference direction.
The currently selected logo uses a stronger green/blue pairing.
**Final production hex values must be extracted from the approved source artwork before export and implementation.**

Until that extraction is done:
- treat the current logo artwork as the visual source of truth
- do not invent alternate “official” hex values without sampling the master logo file

## 7.3 Usage logic
- blue may dominate product/system surfaces
- green should support care and brand warmth
- red should be limited to alerts, sensitive health states, or exceptional status messaging
- neutrals should keep the experience light and readable

## 8. Logo system

## 8.1 Current selected brand direction

The current selected logo direction for PETTODO consists of:

### A. Wordmark
A bold, geometric, highly legible wordmark reading:

**PETTODO**

This wordmark should feel:
- modern
- stable
- simple
- confident
- easy to read at a glance

### B. Brand symbol / favicon
A circular icon using negative space to form an **animal-head silhouette** inside a rounded enclosing shape.

The symbol should communicate:
- pet identity
- protection / containment
- usefulness
- recognizability
- calm trust

This symbol is intended to work as:
- favicon
- app icon
- social avatar
- pin
- sticker
- small brand mark
- secondary badge

## 8.2 Strategic meaning of the symbol
The symbol should be interpreted as:
- a pet/animal identity mark
- a protective, enclosed, memorable sign
- a compact symbol that feels both organic and system-ready

Important:
It should not be over-explained.
A strong brand symbol should be recognizable without needing a long story every time.

## 8.3 Logo-family structure
The PETTODO logo system should include:

1. **Primary full logo**
   - wordmark + symbol

2. **Wordmark-only version**
   - PETTODO without symbol when needed

3. **Symbol-only version**
   - for favicon, app icon, pin, etc.

4. **Monochrome version**
   - black / white or one-color version

5. **Small-size version**
   - simplified if necessary for tiny surfaces

## 10. Logo usage rules

## 10.1 What must remain stable
Do not casually alter:
- the symbol shape
- the wordmark proportions
- the color relationship
- the spacing between symbol and wordmark
- the silhouette logic of the icon

## 10.2 What should be tested before rollout
Before final rollout, verify the logo in:
- white background
- dark background
- monochrome
- one-color print
- favicon size
- app icon size
- small pin/sticker usage
- header/nav usage
- landing hero usage

## 10.3 Gradient rule
If the symbol is ever presented with gradient or richer digital rendering, the logo must still have a **flat-color official version**.

Core principle:
The brand must not depend on gradient effects to work.

---

## 11. What the logo should communicate

The PETTODO logo should communicate, in priority order:

1. trust
2. care
3. utility
4. animal relevance
5. recognizability
6. quiet modernity

It may suggest community indirectly, but community should not be forced into the symbol with literal crowds, many nodes, or noisy social motifs.

---

## 12. What the logo must avoid

The PETTODO logo system must avoid:

- generic paw-only solutions
- heart + paw clichés
- veterinary cross clichés
- obvious AI/circuit/robot tropes
- childish or cartoonish pet illustrations
- excessive detail
- over-complex hidden meanings
- stock-icon feeling
- visual imbalance between text and icon
- unreadable small-size forms
- overly descriptive “search app only” symbolism
- anything that makes PETTODO feel like only a lost-pet finder

---

## 13. Typography direction

The PETTODO wordmark should remain:
- bold
- geometric
- highly legible
- stable
- modern

Until a full typography system is documented:
- treat the current wordmark as locked logo artwork
- do not replace it casually with a random font approximation
- any future typographic refinement must preserve recognizability and clarity

For product/UI typography, a separate typography system may be defined later.
The logo wordmark and the UI font do not need to be identical.

---

## 14. Relationship between logo and product

The logo should support the larger PETTODO product strategy:

- useful before emergencies
- serious when dealing with trust-sensitive actions
- warm enough for everyday care
- broad enough to support owned pets and community dogs
- not limited to one specific feature

The logo should feel valid across:
- management flows
- QR identity flows
- recovery flows
- community-animal flows
- landing / marketing
- future mobile surfaces

## 15. Visual system guidance for product and landing

### Product
The logo should sit inside a UI system that feels:
- calm
- clean
- readable
- blue-forward for trust
- green-supported for care

### Landing
The landing should use the logo as:
- a trust anchor
- a modern startup signal
- a community-care signal
- not as a decorative pet icon only

### Marketing surfaces
Across landing, docs, pitches, banners, and social:
- the symbol should be reusable alone
- the full logo should remain primary when brand recall matters
- the system should feel consistent and not overdesigned

## 16. Photography and illustration direction

The broader brand should prefer:
- real animal photography
- real urban context
- real usefulness
- clean digital layouts

Illustration may be used as support, but PETTODO should not rely on overly cute illustration as the main visual language.

Reason:
The product must feel credible and operational in real life.

## 17. Brand mistakes to avoid in future work

Future design work must avoid these mistakes:

### Strategic mistakes
- making PETTODO look like only a rescue/lost-pet brand
- making PETTODO look like only a services marketplace
- making PETTODO look like a veterinary clinic

### Visual mistakes
- overusing red
- making the symbol too cute
- adding too many explanatory elements
- using weak generic iconography
- allowing logo inconsistency across landing/app/docs

### Production mistakes
- shipping without monochrome version
- shipping without favicon test
- shipping without small-size validation
- using untested color values
- allowing random logo redraws by different tools or teammates
