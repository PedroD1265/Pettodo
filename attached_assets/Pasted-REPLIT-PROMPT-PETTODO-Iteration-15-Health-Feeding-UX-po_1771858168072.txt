P0-2) Global Back Button loop prevention (critical)
Problem: Some flows ping-pong between two screens when pressing back.

Goal:
- Implement a generic safeBack() strategy in AppBar so back never oscillates indefinitely.

Minimum implementation:
- Track last N visited paths (N=4) in sessionStorage (or in-memory if easier).
- On back press:
  1) If history is shallow (idx==0 or no safe back), go to correct mode home:
     - /home-daily if Daily mode
     - /home-emergency if Emergency mode
  2) If loop detected (current path equals path from 2 steps ago OR pattern A->B->A),
     then navigate to a sensible fallback:
     - Daily screens fallback: /home-daily or /daily/pet-list depending on current module
     - Emergency screens fallback: /home-emergency or /emg/report-published if active case
- Keep existing behavior for normal cases.

Also do a quick scan:
- Search for screens using custom backTo props or explicit navigate(-1)
- Fix the worst offenders if you find any obvious ping-pong pairs.

Acceptance:
- Back button cannot bounce between 2 pages more than once; it falls back to a stable parent/home.
