## Minimum persistence rule (must follow)

This task must follow this exact persistence rule:

> **The real file lives in Azure Blob. PostgreSQL stores the durable reference to that file. The UI must only report success when both storage and DB persistence succeed.**

### Therefore:
- do **not** store image binaries in PostgreSQL
- do **not** treat browser preview or local state as persistence
- do **not** treat a Blob upload alone as sufficient if DB persistence failed
- do **not** treat a DB record alone as sufficient if Blob upload failed
- do **not** fake success before the full operation is consistent

### Preferred persistence shape
Use a **small separate image-reference table design** rather than a single fragile `image_url` column if the repo truth supports multiple images or likely future multiple-image use.

Preferred minimal fields:
- `id`
- `pet_id` or `case_id`
- `blob_path` or `storage_key`
- `mime_type`
- `original_filename`
- `size_bytes`
- `is_primary` if useful
- `sort_order` if useful
- `created_at`

Do **not** overbuild a universal media platform.
Keep it minimal and practical.
