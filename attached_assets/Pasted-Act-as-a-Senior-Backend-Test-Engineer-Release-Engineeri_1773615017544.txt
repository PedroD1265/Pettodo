## Deliverables
You must implement all of these:

### 1) Make the Express app testable
Do the smallest safe refactor needed so tests can import the app without startup side effects.

Required result:
- create `server/app.ts`
- `server/app.ts` must:
  - create the Express app
  - configure middleware
  - mount the existing routes
  - export the app
- keep `server/index.ts` as startup-only:
  - import `app`
  - run `testConnection()`
  - run `runMigrations()`
  - call `listen(...)`

Important:
- `app.ts` must not run DB connection tests
- `app.ts` must not run migrations
- `app.ts` must not call `listen`
- importing `app.ts` in tests must have no infrastructure side effects

Do not change route behavior beyond what is strictly needed for testability.

### 2) Add minimal test tooling

Update `package.json` with:
- `test`
- optional `test:watch` only if it is truly useful

Do not add unnecessary libraries.

### 3) Add minimal backend tests
Create a small, stable suite.

Recommended test folder:
- `tests/`

#### A. Health
Test:
- `GET /api/health` returns `200`
- body is `{ ok: true }`

#### B. Auth me
Test both:
- no auth header -> `401`
- valid mocked token path -> `200`

Important:
- do **not** globally bypass `verifyToken`
- mock the smallest auth dependency needed so the real middleware path can still produce both `401` and `200`

Validate response shape:
- `uid`
- `email`
- `displayName`
- `photoURL`

#### C. Pets
Cover:
- `GET /api/pets`
- `POST /api/pets`
- `PUT /api/pets/:id`
- `DELETE /api/pets/:id`

Minimum cases:
- list returns array
- create without `name` -> `400`
- valid create -> `201`
- update existing -> `200`
- update missing -> `404`
- delete existing -> `200`
- delete missing -> `404`

#### D. Cases
Cover:
- `POST /api/cases`
- `GET /api/cases`
- `GET /api/cases/:id`

Minimum cases:
- invalid type -> `400`
- valid create -> `201`
- list returns array
- existing case -> `200`
- missing case -> `404`

#### E. Public pet
Cover:
- `GET /api/public/pet/:petId`

Minimum cases:
- missing pet -> `404`
- existing pet -> `200`
- assert response is public/minimized and does not expose owner data

#### F. Import
Cover:
- `GET /api/import/status`
- `POST /api/import/pets`

Minimum cases:
- status when nothing imported yet
- invalid payload -> `400`
- valid payload -> success
- duplicate import -> `409`

### 4) Mock strategy
Keep it simple.

For auth:
- mock token verification dependency, not the whole endpoint

For DB:
- mock `query`
- mock `pool.connect` only where strictly necessary

Rules:
- no real Firebase
- no real PostgreSQL
- no real secrets
- no real infra mutation
- tests must be deterministic and fast

Do not build a new abstraction layer just for tests.

Do not add deployment.

Only minimal CI.
